# The 5 to 9 — Implementation Plan (v1)

Spec: [`../specs/2026-06-14-the-5-to-9-design.md`](../specs/2026-06-14-the-5-to-9-design.md)
Date: 2026-06-14 · Built autonomously, research-grounded (16-agent research pass + distilled build brief).

This plan is executed as **iterative loops with mechanical test gates between them**
(the project dogfoods its own "build → test → refine scope → continue" cadence).

## Research-locked invariants (must hold in every file)

- **Orchestrator-worker, single writer.** Floor Manager leads; workers are bounded
  and ephemeral. No peer swarm, no live agent-to-agent chat. Coordination is via
  beads on disk.
- **Read-parallel / write-serial.** Fan out read/analysis (QA test-design, security
  scan, research). Serialize writes; isolate only truly-independent writes in
  worktrees with `BEADS_DIR` set.
- **beads = brain, the loop = dumb heartbeat.** Each iteration: `bd ready` → empty?
  emit `QUEUE-EMPTY` → else claim → fresh-context worker does ONE bead (TDD) →
  mechanical gate (typecheck+lint+test+build green) → commit → `bd close`.
- **Fresh process per iteration for long runs** (`night-shift.sh`); in-session
  `/clock-in` is for short watched shifts only (context rot caveat documented).
- **Independent rubric-driven QA.** Producer never grades itself. Pair specs with a
  proof test that fails on a stub (no placeholder cheating).
- **Mechanical backpressure blocks close on red.** No green, no close.
- **Always cap iterations** (default 30). Unlimited loops can't be stopped manually.
- **No-clobber.** Additive context only; read+obey user docs; keep `.beads` local;
  trim any bd-injected CLAUDE.md block. Instruction priority: user repo > 5to9 > defaults.
- **Hard-gate irreversibles** (deploy/publish/force-push/remote-delete/secret-destroy)
  via PreToolUse classifier; everything reversible proceeds.
- **Thin byte-stable init + just-in-time loading + subagent isolation** to fight bloat;
  `bd prime` on SessionStart/PreCompact; right-size models per role.
- **Windows:** pin hook commands to Git Bash (`C:/Program Files/Git/bin/bash.exe`).
